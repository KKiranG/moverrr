# Reference-Only Material

This folder is for small curated reference files that are useful to keep near the repo but are not authoritative.

## What belongs here

- short archived notes or examples worth preserving for future context
- reference snapshots that influenced implementation or review
- intentionally kept material that would be noisy or misleading in canonical docs

## What does not belong here

- live queue state or transactional backlog updates
- imported tooling trees, global skill installs, or vendored workflow libraries
- machine-local research dumps, browser exports, or scratch work
- product or workflow truth that should live in `AUTHORITY.md`, `AGENTS.md`, `CLAUDE.md`, or another canonical doc

## Required labeling

Each new reference file should state:

- source
- why it is being kept
- when it was imported or written
- that it is non-authoritative and cannot overrule canonical docs

If a reference file changes how we understand the product or repo workflow, update the canonical doc in the same work unit instead of leaving the durable truth here.
